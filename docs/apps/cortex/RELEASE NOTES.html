r2.1.2

Bug fixes & minor UI improvements
  • Fixes for several UI bugs (including a severe one exposed by the BeOS 5.0.1 update -- the inability to drag nodes from the Media Add-Ons window.)
  • Errors and confirmation of actions now appear in the status panel in the lower left of the routing window.

  • Bugs fixed, blah blah blah.

 
dr2.0.2
  		[up] [down]
PowerPC Version
  • Cortex now compiles on the Metrowerks PPC compiler.
Features
  • Position reporting: the node inspector window now displays the node's playback position in SMPTE format. Position reporting has been disabled for video-buffer producers that read from files (the video-file players and/or codecs in BeOS R4.5.2 don't take well to repeated querying about their position.)
     
  • Improved tool-tips.

 
dr2.0.1
  		[up] [down]
Fixed Bugs:
  • Broken ToneProducer sawtooth waveform.
  • Groups not split when the sole connection joining them is broken.
  • No default group for video input node(s) (which keeps you from starting them.)
  • Thread-control variables not declared volatile.

B_RECORDING: support for filtering live streams from physical inputs has been vastly improved.

Workaround: on the BeOS R4.5.2, some nodes for audio-input hardware (such as the emu10k) stop responding when requested to stop. Physical-input nodes cannot be stopped for now (once the node has been started, the group inspector 'Start' and 'Stop' buttons will have no effect.)
